# Site Architecture and URL Strategy

## Recommendation

Build ProofSignals as a single unified product with:
- a public marketing site
- a private application
- optional public profile pages
- optional public marketplace pages later

## High-Level Structure

### Public marketing side
Used for:
- homepage
- value proposition
- pricing
- trust/safety
- community rules
- feature overview
- sign up / sign in

### Private application side
Used for:
- onboarding
- AI Asset Builder
- posting assets
- matching
- internal feedback threads
- video feedback workflows
- marketplace management
- profile controls
- moderation workflows

### Optional public profile side
Used for:
- shareable founder reputation pages
- approved public feedback clips
- approved public excerpts
- selected offers / services

---

## Recommended URL Structure

If using a single main domain:

- `/` — homepage
- `/pricing` — pricing
- `/community` — community guidelines
- `/safety` — trust and safety
- `/market` — marketplace landing page (optional public intro)
- `/u/[username]` — public profile page
- `/a/[slug]` — optional public asset page
- `/app` — authenticated application shell
- `/app/dashboard`
- `/app/assets`
- `/app/matches`
- `/app/profile`
- `/app/marketplace`
- `/app/settings/privacy`

If using a dedicated app subdomain:
- `proofsignals.[tld]` — public marketing site + public profiles
- `app.proofsignals.[tld]` — private application

This is often the cleanest split.

## Homepage Messaging Direction

The homepage should lead with:
- trusted founder-to-founder feedback
- meaningful human insight
- AI-assisted asset setup
- optional public proof

Avoid leading with:
- star chasing
- review generation hype
- engagement hacking language

### Suggested homepage message shape
Hero:
- Headline: Honest feedback from founders and marketers who actually get it.
- Subheadline: Post your asset, get meaningful human feedback, build trust, and publish public proof only when you choose.

Sections:
1. How it works
2. AI Asset Builder
3. Human feedback formats
4. Reputation and trust signals
5. Marketplace
6. Optional public profiles
7. Safety and community rules
8. Pricing

## Public Profile Page Structure

Recommended modules:
- Hero card
- About / expertise
- Known for
- Helpful in
- Public feedback excerpts
- Public video clips
- Selected assets or wins
- Services / marketplace offers
- Neutral external presence links

## Profile Privacy Controls

Inside settings, users should control:
- profile visibility
- public indexing
- public video display
- feedback excerpt approval
- public asset display
- external profile links
- hide identity until feedback completed

## Marketplace Visibility

Marketplace can be phased:

### Phase 1
- marketplace visible to members only

### Phase 2
- public landing page describing marketplace
- listings still member-gated

### Phase 3
- selected public offers visible externally

## Why a Unified Architecture Wins

A unified architecture is better than a split-brand architecture because it:
- builds one trust layer
- compounds SEO on one domain
- reduces user confusion
- simplifies product and marketing alignment
- keeps the brand feeling premium and coherent

## Bottom Line

ProofSignals should feel like one product with multiple visibility modes, not two separate brands stitched together.
